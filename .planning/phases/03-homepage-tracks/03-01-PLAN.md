# Phase 3, Plan 01 — Homepage, TrackCard, All-Tracks Page

## Goal

Implement the homepage hero section, the TrackCard component, and the all-tracks page. These are all Server Components — no localStorage, no hooks.

## Tasks

<task id="1">
Create `components/TrackCard.tsx`.

It is a Server Component (no `'use client'`). Accept a `Track` prop. Display:
- Track icon (large, aria-hidden)
- Track title (`text-primary`, bold)
- Track description (`text-muted`)
- Lesson count: "{n} lessons" (`text-muted`)
- Ticket count: "{n} tickets" (`text-muted`)
- The whole card links to `/tracks/{track.id}` via Next.js `<Link>`

Style: `bg-surface`, `border border-subtle`, rounded, padding. No shadows. No inline styles.
</task>

<task id="2">
Replace `app/page.tsx` stub with the homepage.

Server Component. Import `tracks` from `@/data/tracks` and `TrackCard` from `@/components/TrackCard`.

Layout:
- Hero section: headline "MAC Learn", sub-headline "A self-paced backend curriculum for MAC members.", CTA button linking to `/tracks` styled with `accent` background + `bg-base` text
- Below hero: a responsive grid of TrackCard for each track in `tracks`

No `'use client'`. No inline styles. No hardcoded hex.
</task>

<task id="3">
Replace `app/tracks/page.tsx` stub with the all-tracks page.

Server Component. Import `tracks` from `@/data/tracks` and `TrackCard` from `@/components/TrackCard`.

Layout:
- Page heading "All Tracks" (`text-primary`, large)
- Responsive grid of TrackCard for each track

No `'use client'`. No inline styles.
</task>
