# Phase 3, Plan 02 — Track Detail Page, LessonCard, ProgressBar

## Goal

Build the track detail screen: a lesson list with per-lesson completion state and a progress bar. Uses `useProgress` for live state, so requires a Client Component boundary for the lesson list.

## Tasks

<task id="1">
Create `components/ProgressBar.tsx`.

Server-compatible — no `'use client'`. Accept `completed: number` and `total: number` props.

Render:
- A labelled progress bar: "X / Y lessons complete"
- A bar track (`bg-surface`, `border border-subtle`, rounded full, h-2)
- A fill bar (`bg-accent`, rounded full, h-2) with `width` set as `style={{ width: \`${pct}%\` }}` where pct is clamped to [0, 100]
- The inline style for width is acceptable here (dynamic value, not a design token)
</task>

<task id="2">
Create `components/LessonCard.tsx`.

Client Component (`'use client'`). Accept props: `lesson: Lesson`, `isUnlocked: boolean`, `isCompleted: boolean`.

Display:
- Stage badge: "Stage {n}" — small, `text-muted`
- Duration: `text-muted`
- Title: `text-primary`, bold
- Description: `text-muted`, small
- Completion state:
  - If `isCompleted`: green checkmark icon (✓) or "Complete" label in `accent` color
  - If `!isUnlocked && !isCompleted`: muted card style (reduced opacity or `text-muted` throughout); show "Locked" label
  - Otherwise: default card style
- The card links to `/lessons/{lesson.id}` via `<Link>` regardless of lock state (all lessons readable)

Style: `bg-surface`, `border border-subtle`, rounded, padding. No inline styles for design — only width for ProgressBar (see plan 01).
</task>

<task id="3">
Create `components/LessonList.tsx`.

Client Component (`'use client'`). Props: `lessons: Lesson[]`.

- Calls `useProgress()`
- Derives `completedCount` = number of lessons in the list that are completed
- Renders `<ProgressBar completed={completedCount} total={lessons.length} />`
- Renders a list of `<LessonCard>` for each lesson, passing `isUnlocked` and `isCompleted` from `useProgress`
- While not `hydrated`, render a skeleton or null (avoid hydration flash)
</task>

<task id="4">
Create `app/tracks/[trackId]/page.tsx`.

Server Component. Needs `generateStaticParams()`.

```ts
export function generateStaticParams() {
  return tracks.map(t => ({ trackId: t.id }))
}
```

- Look up the track by `trackId` param; if not found, call `notFound()` from `next/navigation`
- Filter `lessons` array to only those in `track.lessons` (maintain order from `track.lessons`)
- Render:
  - Track icon + title + description
  - `<LessonList lessons={trackLessons} />` (Client Component)
  - Ticket section: list of ticket IDs from `track.tickets` as links to `/tickets/{id}`

Import `tracks` from `@/data/tracks`, `lessons` from `@/data/lessons`.
</task>
