# Phase 4, Plan 01 — Lesson Detail Page

## Goal

Implement `ResourceLink`, `MarkLessonComplete`, and the full lesson detail page at `/lessons/[id]`.

## Tasks

<task id="1">
Create `components/ResourceLink.tsx`.

Server-compatible (no `'use client'`). Accepts `resource: Resource` prop (import type from `@/types`).

Renders:
- An anchor tag that opens in a new tab (`target="_blank" rel="noopener noreferrer"`)
- The resource title as link text (`text-primary`)
- A type badge showing the resource type:
  - `video` → badge with `bg-accent text-base text-xs font-medium`
  - `reading` → badge with `border border-subtle text-muted text-xs`
  - `interactive` → badge with `border border-subtle text-accent text-xs`
- The whole element is styled with `bg-surface border border-subtle rounded-lg p-4` with flex layout

The badge labels: `video` → "Video", `reading` → "Reading", `interactive` → "Interactive".
</task>

<task id="2">
Create `components/MarkLessonComplete.tsx`.

**CoVe required** (useState, useEffect via useProgress, conditional render).

Client Component (`'use client'`). Props: `lessonId: string`.

CoVe Stage 1 — Write code [UNVERIFIED]:
- Import `useProgress` from `@/hooks/useProgress`
- Call `const { completeLesson, isLessonCompleted, hydrated } = useProgress()`
- Derive `completed = isLessonCompleted(lessonId)`
- While `!hydrated`, render a placeholder button (disabled, same size, `text-muted border border-subtle`) to avoid layout shift
- When hydrated and `completed`: render a non-interactive "✓ Completed" state in `text-accent`
- When hydrated and not completed: render a button that calls `completeLesson(lessonId)` on click, styled with `bg-accent text-base font-semibold px-6 py-3 rounded-md hover:opacity-90 transition-opacity`

CoVe Stage 2 — Verification targets:
1. `hydrated` guard prevents rendering interactive button during SSR/before hydration
2. `completeLesson` is only called once — idempotent (hook already guards this, but confirm)
3. Button re-renders to completed state without page reload after click
4. No crash if `lessonId` is not in the data

CoVe Stage 3 — Verify each:
1. `!hydrated` branch returns early with placeholder — confirmed safe for SSR
2. `useProgress.completeLesson` uses `setCompletedLessons(prev => { if (prev.includes(lessonId)) return prev; ... })` — idempotent ✓
3. State update via `setCompletedLessons` triggers re-render — `isLessonCompleted` returns true on next render ✓
4. `isLessonCompleted` just checks array inclusion — no crash on unknown ID ✓

CoVe Stage 4 — [VERIFIED]: proceed to implementation.
</task>

<task id="3">
Create `app/lessons/[id]/page.tsx`.

Server Component. `generateStaticParams()` returns all lesson IDs.

```ts
export function generateStaticParams() {
  return lessons.map(l => ({ id: l.id }))
}
```

Props: `{ params: { id: string } }`.

Lookup: find lesson by `params.id`; if not found, call `notFound()`.

Layout (top to bottom):
1. Breadcrumb: `← {track.title}` link back to `/tracks/{lesson.trackId}` — use `text-muted hover:text-primary text-sm`; look up track from `data/tracks.ts` by `lesson.trackId`
2. Metadata row: Stage badge (`text-muted text-xs uppercase tracking-wide`) + duration (`text-muted text-xs`)
3. Title: `text-3xl font-bold text-primary mb-2`
4. Description: `text-muted mb-8`
5. Skills section — heading "You'll learn", `ul` with each skill as `li` with a small accent bullet (`text-accent mr-2`), `text-primary text-sm`
6. Resources section — heading "Resources", list of `<ResourceLink>` per resource in `lesson.resources`
7. Checkpoint section — heading "Self-check", `bg-surface border border-subtle rounded-lg p-5` block containing the `lesson.checkpoint` text in `text-muted`
8. `<MarkLessonComplete lessonId={lesson.id} />` at the bottom

Wrap the content in `<div className="mx-auto max-w-3xl px-4 sm:px-6 py-12">`.
</task>
